添加库的使用需求

练习一：添加库的是使用需求

通过目标参数的使用要求可以更好的控制库或执行程序的链接和头文件引入，同时也可以对CMake中目标的传递属性进行更多的控制。用到的主要命令如下：

  • target_compile_definitions()
  • target_compile_options()
  • target_include_directories()
  • target_link_directories()
  • target_link_options()
  • target_precompile_headers()
  • target_sources()

解决方案

让我们重构上一步代码，来使用CMake的现代化使用需求方法。

我们希望声明，任何链接到MathFunctions的库或运行程序都包含当前源码目录，而MathFunctions本身不需要包含。我们可以使用 INTERFACE 表示使用需求。记住 INTERFACE 意味着使用者需要而本身不需要。

在 MathFunctions/CMakeLists.txt 文件后面，使用带有INTERFACE关键字的 [target_inlcude_directories()](https://cmake.org/cmake/help/latest/command/target_include_directories.html#command:target_include_directories) 方法，如下：

target_include_directories(MathFunctions INTERFACE ${CMAKE_CURRENT_SOURCE_DIR})

既然我们已经指定了 MathFunctions 的使用要求，我们可以 CMakeLists.txt 文件头部安全的删除 EXTRA_INCLURE变量的使用了。

TODO 2: CMakeLists.txt
if(USE_MYMATH) add_subdirectory(MathFunctions) list(APPEND EXTRA_LIBS MathFunctions) endif()

TODO 3: CMakeLists.txt
target_include_directories(Tutorial PUBLIC "${PROJECT_BINARY_DIR}" )

使用这种方法，我们的可执行程序目标引用库只需要通过库的名称调用 target_link_libraries() 方法。在大型项目中，使用经典方法手动指定库依赖项会很快使项目变得非常复杂。
