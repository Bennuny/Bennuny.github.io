步骤一：起点

应该从哪里开始学习CMake？这篇教程介绍一下CMake的基本语法、命令和变量。了解这些概念之后，我们通过三个练习创建一个简单的CMake项目。

这个教程的每个练习会先给出练习的背景信息。然后给出一个目标和有帮助资源列表。在 编辑文件 部分下的每个文件在 Step1 目录下，包含一个或多个 TODO 注释。 每个 TODO 注释代表需要添加或者修改一行或两行代码。这些 TODO 要按顺序完成，先完成 TODO1 然后 TODO2 等。在 入门 部分给出一些有帮助的提示并指导你完成练习。然后 构建和运行 部分会一步步的告诉你如何构建和测试本次练习，在每个练习结束时，讨论预期的解决方案。

练习一：构建基本项目

大多数CMake的基础项目都是从一个单源码文件构建一个可执行程序开始。对于一个这样的简单项目，只需要在 CMakeLists.txt 中添加三行命令。

注意： 尽管CMake支持大写、小写和大小写混合，这篇教程首选小写字母。

任何项目的CMakeList.txt文件开头要指定兼容的CMake最小版本号，使用 cmake_minimum_required() 命令指定。这个策略设置保证接下来的CMake方法运行在可兼容的CMake版本。

项目开始，我们使用 project() 命令设置项目名称。每个项目都需要这个命令，并且应该在 cmake_minimum_require() 后紧跟调用。正如我们稍后看到的，这个命令也可以用来指定其他项目的一些信息，例如语言或版本号等。

最终，add_executable() 命令告诉CMake用指定的源文件构建可执行程序。

练习二：指定C++标准

CMake有一些特殊的变量，要么在之后的场景中指定，或对于CMake构建项目代码有实际意义。这些变量其中一些以 CMAKE_ 开头。在项目中创建变量时避免使用此命名约定。其中两个特殊变量是：CMAKE_CXX_STANDARD和CMAKE_CXX_STANDARD_REQUIRED。这两个变量一起指定了构建项目的C++标准。

练习三：增加版本号和配置头文件

如果可以在 CMakeList.txt 中定义一个变量，在代码中也可以访问这个变量，那是非常有帮助的。在这个例子中，我们将要打印项目版本号。

一个解决方法是配置一个头文件。创建一个输入文件，包含一个或多个可被替代的变量。通过一个特定的语法定义这些变量，@VAR@. 然后，通过 configure_file() 命令将输入文件复制到指定输出文件，并用CMakeLists.txt 中的 VAR 当前值替换这些变量。

我们可以在源码中直接编辑版本号，但首选使用这个方法定义版本号，因为它创建了单一真实来源并避免了重复定义。

目标

定义并报告项目版本号。

资源列表

  • <PROJECT-NAME>_VERSION_MAJOR
  • _VERSION_MINOR
  • configure_file()
  • target_include_directories()

编辑文件

  • CMakeLists.txt
  • tutorial.cxx

解决方案

本次练习，改善我们的可执行程序输出版本号。虽然我们可以在源码中实现，但使用 CMakeLists.txt 可以确保维护版本号的单一数据源。

首先，修改 CMakeLists.txt 文件，使用 project() 命令设置项目名称和版本号。当调用 project() ，CMake在幕后定义了 Tutorial_VERSION_MAJOR 和 Tutorial_VERSION_MINOR。

TODO 7: Click to show/hide answer
TODO 7: CMakeLists.txt
project(Tutorial VERSION 1.0)

然后调用 configure_file() 拷贝输入文件并用CMake替代指定变量。

TODO 8: Click to show/hide answer
TODO 8: CMakeLists.txt
configure_file(TutorialConfig.h.in TutorialConfig.h)

由于配置文件会被写入项目的运行目录，因此需要将该目录添加到路径列表以搜索包含文件。

注意：本教程中，我们可互相引用项目的构建和项目的二进制目录，这些都是相同的，并不是指bin/目录。

使用 target_include_directories() 指定可执行目标在何处查找头文件。

TODO 9: Click to show/hide answer
TODO 9: CMakeLists.txt
target_include_directories(Tutorial PUBLIC "${PROJECT_BINARY_DIR}")

TutorialConfig.h.in 是要配置的输入文件。当在 CMakeLists.txts中 调用 configure_file()，变量 @Tutorial_VERSION_MAJOR@ 和 @Tutorial_VERSION_MINOR@ 会被替换到 TutorialConfig.h 中项目对应版本号。

TODO 10: TutorialConfig.h.in
#define Tutorial_VERSION_MAJOR @Tutorial_VERSION_MAJOR@
#define Tutorial_VERSION_MINOR @Tutorial_VERSION_MINOR@

接下来，我们修改 tutorial.cxx 文件，引入配置头文件 TutorialConfig.h。

TODO 11: Click to show/hide answer
TODO 11: Tutorial。cxx
#include "TutorialConfig.h"

最后，在 tutorial.cxx 中输出可执行程序的名字和版本号：

TODO 12: Tutorial。cxx
if (argc < 2) { // report version std::cout << argv[0] << " Version " << Tutorial_VERSION_MAJOR << "". << Tutorial_VERSION_MAJOR << std::endl; std::cout << "Usage: " << argv[0] << " number" << std::endl; return 1; }
